# Fresh Glass UI 重构

## Goal

在不改变三消引擎和控制器时序的前提下，使用素材子任务产出的六张最终 PNG，
完成 Fresh Glass 展示契约、响应式布局、交互状态和无障碍样式重构。

## Dependency

仅在 `07-17-fresh-glass-candy-assets` 完成并通过素材验证后启动。本任务不得
打开生成源图、重新切图或调用图片生成。

## Requirements

- 将字符 glyph 展示字段迁移为本地素材 URL，文本无障碍标签保持不变。
- 重建蓝紫背景、玻璃外围面板、深蓝棋盘、按钮、焦点、弹窗和 reduced-motion
  token，清除旧纸张/印章视觉语言。
- 桌面采用顶部工具区、左 HUD、中央棋盘、右操作/说明；移动端按 HUD → 棋盘
  → 操作/说明排列，320px 起无横向滚动。
- 棋盘始终正方形，六类棋子及选择、焦点、无效交换、消除、生成/下落和洗牌
  状态不能只依赖颜色。
- 保留鼠标、触控、键盘、焦点恢复、实时播报和控制器阶段时序。
- 每轮只完成一个子系统并运行其定向检查；浏览器多视口截图留给 QA 子任务。

## Acceptance Criteria

- [ ] 六类棋子均通过本地 `<img>` 素材渲染，不再出现字符 glyph。
- [ ] 1440px、768px、390px 和 320px 布局规则均由 CSS 明确定义，棋盘正方形且
  无横向溢出。
- [ ] 交互、焦点、状态动画、弹窗和 reduced-motion 保持完整。
- [ ] 触及文件不再引用旧视觉 token 或旧纸张/印章文案。
- [ ] 定向 UI 测试、lint、typecheck、单元测试和 production build 通过。
- [ ] 未重新打开或生成素材源图；未修改引擎规则和公共契约。

## Out of Scope

- 素材制作、全视口截图对比、`design-qa.md` 和新的游戏进程功能。
