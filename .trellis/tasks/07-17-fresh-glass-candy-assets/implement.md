# Candy 素材整理实施计划

## Checkpoints

1. 将 4 个源文件的名称、尺寸、模式、alpha 状态写入 inventory；不查看图片。
2. 为第 1 张未确认源图生成 <=512px 预览，只检查该图并记录可用内容。
3. 重复步骤 2，每轮只推进一张尚未确认的源图。
4. 每轮只提取并验证一个 `candy-{type}.png`，成功后立即写入 inventory。
5. 六张输出齐备后运行统一 alpha、边界、尺寸和文件存在性检查。
6. 删除无长期价值的临时预览，完成任务检查。

## Stop rules

- 不打开原尺寸源图。
- 不在一次工具调用中查看多张图片。
- `violet`、`rose` 已各生成并选定一张；不再调用图片生成。
- 不修改 UI 代码。
- 若当前检查点中断，下轮从 inventory 中第一个未完成条目继续。

## Validation

运行最小素材验证脚本或等价命令，输出每个文件的尺寸、色彩模式、alpha 和
非透明边界；本子任务不运行全量 UI 测试。
