# 猫咪默认摆位与抚摸寻鱼交互

## Goal

让猫咪默认待在花盆旁边，并把“点击猫咪立刻寻鱼”的突兀行为改成先与猫咪互动、再明确选择是否请它帮忙抓鱼。

## Background

- 猫咪的默认位置由 `GameView.vue:353` 的场景槽位控制，桌面与移动端分别设置了不同的水平位置。
- 当前猫咪按钮在 `CatCompanion.vue:45` 发出单一 `activate` 事件，并在 `GameView.vue:260` 直接绑定到 `requestCatSearch`；因此鼠标、触摸或键盘激活猫咪都会立即进入寻鱼流程。
- 寻鱼、移动、守鱼和返回的现有状态机位于 `ambient-controller.ts:367`，喂鱼仍是独立的拖放或键盘路径。
- 既有产品约束要求猫咪只协助定位合法隐藏鱼，不自动选择、移除或喂食目标。

## Requirements

- R1. 猫咪在 home 状态下默认显示在花盆旁边；桌面、移动端和紧凑横屏都保持自然间距，不遮挡花盆、托盘或主要找鱼区域。
- R2. 直接激活 home 状态的猫咪不得立即启动寻鱼。
- R3. 激活猫咪后应在猫咪旁边出现轻量的双选项互动气泡，提供“摸一下”和“帮我抓鱼”两个明确动作；点击外部或按 Escape 可关闭。
- R4. “摸一下”只触发亲昵反馈，不选择鱼、不移动猫咪，也不修改持久化游戏状态。
- R5. 只有明确选择“帮我抓鱼”才调用现有寻鱼状态机；既有合法目标筛选、休息/吃饱拒绝、节流、移动、守鱼和返回规则保持不变。
- R6. 鼠标、触摸和键盘都能完成相同互动；焦点状态、可访问名称、状态播报和 reduced-motion 反馈保持完整。
- R7. 拖鱼喂猫和聚光灯找鱼等既有玩法保持不变。

## Acceptance Criteria

- [x] AC1. 首次进入或猫咪返回 home 后，猫咪视觉上紧邻花盆，且在 1440×900、390×844、320×568 和紧凑横屏下不发生关键遮挡。
- [x] AC2. 单击、轻触、Enter 或 Space 激活猫咪时，只打开互动入口，不会改变 `catTravelPhase` 或 `guardedPieceId`。
- [x] AC3. 选择“摸一下”会给出可见且可播报的亲昵反馈，猫咪保持 home，游戏与持久化状态不变。
- [x] AC4. 选择“帮我抓鱼”后，现有寻鱼流程照常进入 looking、travelling、guarding，并继续遵守所有拒绝条件。
- [x] AC5. 互动气泡可用键盘进入、选择和退出；点击外部或按 Escape 关闭后焦点回到猫咪，reduced-motion 下仍保留等价状态反馈。
- [x] AC6. 既有拖放喂鱼、键盘喂鱼、守鱼目标解析和场景恢复测试不回归。

## Out of Scope

- 不修改鱼生成、匹配、托盘结算、喂鱼容量或植物成长规则。
- 不新增任务、奖励、饥饿、养成数值或持久化的抚摸次数。
