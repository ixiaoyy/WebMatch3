# 改善鱼块生成难度

## Goal

避免同种鱼以三只一组直接堆在相邻位置，让玩家需要观察和规划不同位置的鱼，而不是连续点击眼前的一组即可消除。

## Background

- 当前关卡生成器把每连续 3 个棋子定义为同一种鱼，同时又把这 3 个棋子放进同一个空间槽位；因此“同种鱼堆叠”不是偶发随机结果，而是生成算法的稳定行为（`apps/web/src/features/game/engine/pile.ts:190-212`）。
- 每关鱼总数保持为 3 的倍数，各鱼种数量也需要保持为 3 的倍数，才能保证整关可完成（`apps/web/src/features/game/engine/ambient-game.test.ts:235-265, 307-313`）。
- 随机源可注入，现有测试使用固定种子；调整后的布局也必须可复现。

## Requirements

### R1. 解耦鱼种与空间分组

- 生成器不得再把一个完整的同种三连组直接映射到同一空间槽位。
- 同一空间槽位内的 3 只鱼必须属于 3 个不同鱼种，让玩家跨位置寻找三连组合。

### R2. 保留关卡不变量

- 不改变现有每关鱼总数、层数和可用鱼种数量的递进配置。
- 每种实际生成的鱼数量仍为 3 的倍数，保证理论上可以全部消除并正常进入下一关。
- 不改变棋子 ID、几何模板、持久化结构和随机源契约。

### R3. 确定性与有限生成

- 相同固定随机种子必须得到相同鱼种与位置组合。
- 分配算法不得依赖无上限重试；如使用约束随机，必须存在明确的有限兜底。

## Acceptance Criteria

- [x] AC1：任一新关卡的每个空间槽位内，3 只鱼都属于不同鱼种。
- [x] AC2：1–8 关使用固定随机种子生成时，每种鱼的总数均为 3 的倍数，且可沿合法消除路径清空并进入下一关。
- [x] AC3：相同种子重复生成得到完全相同的鱼种顺序和空间布局。
- [x] AC4：现有关卡鱼总数、层数、鱼种递进、ID 唯一性及坐标唯一性保持不变。
- [x] AC5：新增定向单元测试能在旧算法上失败，并覆盖槽位多样性与整关可完成性。

## Out of Scope

- 调整托盘容量、三只消除规则、喂猫规则或失败恢复机制。
- 改动鱼图片、动画、灯光和界面布局。
- 重写遮挡/可选择规则或调整关卡数量曲线。
