# 三消核心规则引擎

## Goal

实现与 DOM、Vue 组件和动画解耦的确定性三消规则引擎，作为所有局内交互、计分和后续扩展的唯一规则来源。

## Background

- `07-17-project-foundation` 已完成，当前具备 TypeScript 严格模式、Vitest 与根级 `pnpm ci:web` 验证命令。
- 核心代码归属 `apps/web/src/features/game/engine`，不得放入 `shared` 或 Vue 组件。
- 棋盘采用不可变公开契约；内部结算可使用临时可空矩阵，但不得把中间态泄漏给调用方。

## Requirements

### R1. 棋盘模型

- 支持可配置行列数和棋子种类，首版建议默认 8×8、6 种普通棋子。
- 每个棋子具有稳定唯一标识和类型；表现层状态不得污染规则判断。
- 坐标、交换、匹配和结算结果使用显式 TypeScript 类型。

### R2. 初始生成

- 初始棋盘不得包含横向或纵向 3 个及以上同类棋子。
- 初始棋盘必须至少存在一个可形成匹配的合法相邻交换。
- 随机源可注入或设置种子，测试不得依赖不可复现的 `Math.random()` 结果。
- 生成过程需要有上限与兜底，不能在极端随机序列下无限循环。

### R3. 交换与匹配

- 只接受上下左右相邻坐标的交换；越界、同格和斜向交换均返回明确失败原因。
- 交换后未产生匹配时，规则结果要求表现层复位，且不进入消除流程。
- 识别所有横向和纵向 3 个及以上连续同类棋子。
- 同一棋子同时属于横向和纵向匹配时只能出现一次，但匹配组信息应可供计分使用。

### R4. 消除、坠落与补充

- 一轮结算包含：标记匹配 → 移除 → 每列稳定向下压缩 → 顶部补充。
- 保留列内未消除棋子的相对顺序。
- 新棋子从顶部补充，返回足够的变化描述供 UI 播放消除和坠落动效。
- 坠落产生新匹配时继续结算，直到棋盘稳定；返回连锁轮次与每轮匹配信息。
- 结算过程必须设置最大连锁保护，超过保护阈值返回可诊断错误而非无限循环。

### R5. 可走性与洗牌

- 能判断稳定棋盘是否存在至少一个合法移动。
- 无合法移动时洗牌现有棋子，优先保持棋子类型数量不变。
- 洗牌结果不得自带匹配，且必须包含合法移动。
- 达到洗牌尝试上限后允许重新生成，但需要显式返回发生了兜底重建。

### R6. 输出契约

- 规则函数不操作定时器、DOM、浏览器存储或 Vue 响应式对象。
- 每次操作返回结构化结果，至少能区分：非法坐标、非相邻、无匹配复位、有效匹配、连锁完成、已洗牌。
- 输入棋盘是否原地修改必须形成统一约定；首版优先不可变更新，避免动画层读到中间态。
- 生成、补充和洗牌所需随机源由调用方显式传入；运行时可选择系统随机源，测试必须使用固定种子或序列。

## Acceptance Criteria

- [x] AC1：固定随机源可稳定生成无初始匹配且至少有一步可走的棋盘。
- [x] AC2：同格、越界、斜向和非相邻交换均被拒绝，且不改变原棋盘。
- [x] AC3：无效相邻交换返回复位结果；有效交换返回完整匹配坐标。
- [x] AC4：横向、纵向、4+ 长连、T/L/十字交叉和多个同时匹配均被正确识别并去重。
- [x] AC5：消除后每列稳定坠落并正确补充，连续匹配可结算到稳定状态。
- [x] AC6：无解棋盘可被识别并洗牌为“无初始匹配且有合法移动”的棋盘。
- [x] AC7：随机生成和连锁结算存在有限上限，不会无限循环。
- [x] AC8：核心规则可在无浏览器环境中运行，所有边界用例通过单元测试。

## Dependencies

- 依赖 `07-17-project-foundation` 提供 TypeScript 与测试运行环境。

## Out of Scope

- 分数、步数、胜负和本地存储。
- 动画、声音、视觉主题和组件布局。
- 炸弹、彩虹球、障碍物和特殊棋子组合。
