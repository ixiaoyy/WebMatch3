# Fresh Glass 视觉验收

## Goal

在 UI 重构及自动化检查通过后，按单视口检查点完成浏览器截图、参考方向对比、
交互状态验收和 `design-qa.md` 缺陷收敛。

## Dependency

仅在 `07-17-fresh-glass-ui-restyle` 完成后启动。不得重新生成或重新切分糖果素材。

## Requirements

- 首次只创建一张最长边不超过 1024px 的参考图 QA 副本；禁止打开原尺寸参考图。
- 每轮只检查一个视口：320×568、390×844、768×1024、1440×900。
- 每张截图检查后立即把构图、溢出、棋盘比例、层级、可读性和状态问题写入
  `design-qa.md`，不得等待四张全部打开后再记录。
- 交互状态按选择、焦点、无效交换、消除/下落、洗牌、重开和结果弹窗逐项记录。
- 所有 P0/P1/P2 缺陷必须修复并重新检查对应视口；P3 可记录后交付。
- 浏览器控制台不得有 Vue 警告、资源 404 或未处理异常。

## Acceptance Criteria

- [ ] 四个规定视口各有独立验收记录且无横向溢出，棋盘保持正方形。
- [ ] 1440×900 与参考方向在层级、色彩、玻璃面板、棋盘和糖果质感上明显一致，
  且没有复制 Logo、关卡或虚假道具。
- [ ] 主要交互和无障碍状态均有证据，reduced-motion 与 blur fallback 可读。
- [ ] 所有 P0/P1/P2 已关闭，`design-qa.md` 标记 `final result: passed`。
- [ ] 全程没有批量打开多张原图，也没有重新生成素材。

## Out of Scope

- 新功能、素材重做、布局架构重写或与视觉缺陷无关的代码清理。
