# Fresh Glass Candy 视觉重做

## Goal

以仓库内 `fresh-glass-candy-reference.png` 为唯一视觉源，将旧版纸张/印章
界面重做为清爽、现代的玻璃糖果游戏界面，同时保持已经完成的三消规则、
输入方式、结算时序和无障碍语义不变。

## Background

- 参考图尺寸为 1491×1055，主要特征为蓝紫雾化背景、顶部轻量工具区、中央
  深蓝高对比棋盘、两侧玻璃信息栏和高饱和立体糖果。
- `Sweet Gems` Logo、`Level 24`、具体目标数值和道具库存不是产品需求，
  不得照搬或伪造功能。
- 当前实现仍是暖纸张、大衬线标题、硬边框面板和字符/CSS 印章棋子，已经被
  项目视觉合同标记为旧视觉。
- 当前为可玩的练习模式；计分、有限步数和结果状态由独立任务接入，本任务
  不得提前实现。
- 最初已有素材固定来自
  `C:\Users\phpxi\.codex\generated_images\019f6edd-116b-7331-bf97-afb671a655f6`：
  共 4 张 1254×1254 RGB PNG，当前均无透明通道。
- 用户随后明确批准仅补生成缺失的 `violet` 紫花与 `rose` 粉心各一张；两张
  选定源图已复制到素材子任务的 `research/sources/`。不得再次生成变体。

## Delivery Map

本任务改为只负责跨子任务需求与最终集成，不再作为直接实现目标：

1. `07-17-fresh-glass-candy-assets`：只整理现有 4 张源图，产出六类透明糖果素材。
2. `07-17-fresh-glass-ui-restyle`：只使用最终素材完成展示契约、布局与样式重构。
3. `07-17-fresh-glass-visual-qa`：只做分视口截图、视觉对比与缺陷收敛。

依赖顺序为 1 → 2 → 3；每个子任务必须独立验证并归档后再进入下一项。

## Requirements

### R1. 视觉源与品牌边界

- 桌面构图复现参考图的层级：轻柔背景 → 玻璃外围控件 → 深色棋盘 →
  高饱和糖果。
- 使用项目自己的简洁标题与中文文案，不复制参考图的产品名或关卡品牌。
- 不新增虚假道具、音效设置、关卡或进度控件；右栏只放现有真实操作。

### R2. 页面与响应式布局

- 桌面端使用顶部工具区、左侧 HUD、中央棋盘和右侧操作/玩法区。
- 平板允许信息区收拢，但棋盘仍为第一视觉优先级。
- 320px、390px 手机端按 HUD → 棋盘 → 操作/说明垂直排列，不得只是缩小
  桌面三栏，也不得产生横向滚动。
- 棋盘在所有断点保持正方形，现有 8×8 引擎尺寸继续作为网格来源。

### R3. 视觉系统

- 全局 token 统一定义蓝紫背景、玻璃表面、深蓝棋盘、主操作色、奖励色、
  文字色、模糊、圆角和阴影；不得保留旧 `paper/ink/graphite/acid` 语言。
- 使用现代系统无衬线字体，不使用旧版大衬线展示字体或编辑式眉题。
- 玻璃只用于外围 UI 和弹窗，棋盘保持稳定高对比；不支持
  `backdrop-filter` 时仍有清晰回退。

### R4. 糖果棋子与状态

- 六类棋子使用项目内真实栅格素材，呈现高饱和、圆润、带高光与立体边缘的
  糖果/宝石质感；不再使用字符图形或旧印章纹理。
- 每类棋子同时具有不同色相和持久轮廓：红色圆角方糖、黄色水滴、绿色叶片、
  蓝色圆糖、紫色花糖、粉色心形。
- 选中、键盘焦点、无效交换、匹配、下落/生成和洗牌状态不能只依赖颜色。
- 整理既有 4 张源图，并使用用户明确批准后各生成一次的 `violet`、`rose`
  源图；六张来源确定后不得再次生成。源图检查、素材处理和 UI 实现不得出现
  在同一个执行单元中。

### R5. 交互与可访问性

- 保留鼠标、触控、方向键、Enter/空格、Escape 和焦点管理。
- 所有真实按钮保持至少 44×44 CSS 像素点击区域和明显焦点环。
- 文字满足 WCAG AA；装饰高光不进入无障碍树。
- 减少动态模式移除弹性和大位移，但保留静态状态反馈、播报与完整结算时序。

### R6. 变更边界

- 不改变引擎公共契约、交换规则、随机源、棋盘状态或控制器动画顺序。
- 不引入 UI 框架、全局状态库、路由、后端或远程资源依赖。
- 可调整展示类型和组件 props 来接入本地图片，但展示组件不得推断规则。

### R7. 防中断执行边界

- 不直接查看多张原尺寸图片；先读取尺寸、色彩模式、alpha 和文件大小等元数据。
- 必须视觉检查时，先生成最长边不超过 512px 的临时预览，一次只检查一张，
  检查结论立即写入任务文件。
- 每轮只完成一个可落盘交付物；不得在同一轮串联素材理解、切图、页面实现和
  多视口 QA。
- 临时预览放在任务 `research/` 或 `.tmp/`，最终验收后删除或明确标注用途。
