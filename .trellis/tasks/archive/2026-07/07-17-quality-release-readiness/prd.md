# 质量验证与发布准备

## Goal

以可重复的自动化检查和定向浏览器验收，证明 Web 消消乐 MVP 的规则正确、交互稳定、响应式可用，并达到可部署为静态站点的状态。

## Requirements

### R1. 核心规则测试

- 覆盖初始生成、合法移动检测、各种非法交换、横纵匹配、4+ 长连、交叉去重、多组同时匹配、坠落、补充、连锁和洗牌。
- 使用固定随机源或种子，不允许概率性通过。
- 包含生成/洗牌上限与异常兜底测试，证明不会无限循环。
- 对计分、扣步、胜负和本地存储迁移/损坏回退做独立单元测试。

### R2. 组件与流程测试

- 验证选中/取消/切换选择、无效交换复位、结算时输入锁、重开确认和结果后再来一局。
- 至少一条浏览器流程从开局完成到胜利或失败；可通过固定棋盘保证稳定。
- 测试不得依赖脆弱的动画等待，优先等待可观察状态或使用可控动画模式。

### R3. 响应式与兼容性

- 定向检查 320×568、390×844、768×1024、1440×900。
- 覆盖 Chromium；发布目标确认后再决定是否把 WebKit 与 Firefox 设为必过。
- 断言无横向溢出、棋盘格可点击、结果层可见、固定控件不遮挡内容。
- 触屏与鼠标流程至少各有一条验证路径。

### R4. 可访问性

- 自动检查基础语义、名称、对比度和明显的 ARIA 错误，并做键盘人工流程验收。
- 键盘可完成完整主要流程，焦点不会困在棋盘或结果层之外。
- 减少动态模式下不播放大幅位移/缩放，且状态仍可理解。
- 棋子在灰度或常见色觉差异下仍可通过形状/纹理区分。

### R5. 性能与稳定性

- 棋盘交互不因重复创建全局监听器或未清理定时器而逐局变慢。
- 普通对局中无未捕获异常、Vue 警告或明显布局抖动。
- 生产包保持与纯前端小游戏相称；若新增大型依赖，需要记录理由与体积影响。
- 页面在中端移动设备上保持流畅，动效优先使用 transform/opacity。

### R6. 发布准备

- `lint`、`typecheck`、单元测试、浏览器测试和 production build 具备统一命令。
- 静态构建产物直接可由普通 Web 服务器托管，刷新根路径不依赖后端重写。
- README 记录运行、检查、构建和静态预览方式。
- 使用与 ParallelLines 一致的 Node 22 多阶段构建与 Nginx 静态运行镜像，由独立 Docker Compose 项目部署到 `46.38.157.90`。
- 部署必须使用独立目录、容器名和空闲端口；在未确认现有反向代理边界前不得修改 ParallelLines/FableSpace 的 80/443 服务。
- 生产发布后从外部访问线上地址完成一次核心流程和静态资源回归；不购买域名。

## Acceptance Criteria

- [ ] AC1：lint、typecheck、核心单元测试、关键浏览器流程和 production build 一次通过。
- [ ] AC2：所有规则边界测试使用确定性输入，重复运行结果一致。
- [ ] AC3：指定四类视口无横向溢出、遮挡或不可操作棋子。
- [ ] AC4：鼠标、触屏和键盘主要流程均完成定向验证。
- [ ] AC5：减少动态模式、损坏本地存储和无合法移动等降级路径可用。
- [ ] AC6：浏览器控制台无未处理异常和 Vue 运行时警告。
- [ ] AC7：构建产物可由本地静态服务器启动并完成一局。
- [ ] AC8：独立容器部署到 `46.38.157.90` 后可通过公开地址游玩，且 ParallelLines/FableSpace 现有容器与 80/443 服务保持健康。

## Dependencies

- 在工程骨架、核心引擎、可玩 UI 和局内进度任务完成后执行。

## Out of Scope

- 新购生产域名、HTTPS、CDN、监控和告警；若未提供域名，默认使用服务器 IP 与独立端口。
- 大规模用户测试、A/B 测试和运营数据分析。
- 原生设备实验室与低端机型全量兼容矩阵。
