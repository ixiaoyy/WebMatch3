# Web 消消乐 MVP

## Goal

交付一款打开网页即可开始、规则清晰、单局时长短、桌面与手机均可顺畅游玩的经典三消游戏 MVP，并为后续关卡、特殊棋子和在线能力留下清晰的扩展边界。

`WebMatch3` 是暂定工程名，含义是“网页版三消游戏（Web Match-3）”，不是对外产品名。

## Background

- 新项目位于 `D:\work\WebMatch3`，与 `D:\work\ParallelLines` 同级。
- 工程组织参考 ParallelLines 的 `pnpm workspace → apps/web → Vue 3 + Vite + TypeScript → app / features / shared` 分层方式。
- 只复用工程组织与质量基线，不复制 ParallelLines 的论坛业务、品牌资产、后端服务或部署配置。
- 当前实现按子任务推进；最终发布目标为 `46.38.157.90` 上独立的 Docker/Nginx 静态服务，工程配置参考 ParallelLines 但不得干扰其容器与 80/443 入口。

## Product Principles

1. **即开即玩**：首版不要求注册、登录或联网，进入主页面即可开始游戏。
2. **规则可预期**：交换、消除、连锁、计分和胜负反馈必须明确，不通过隐藏规则制造挫败。
3. **短局体验**：目标单局约 2–5 分钟，适合碎片时间。
4. **触屏优先、桌面完整**：手机不应是压缩后的桌面棋盘；桌面也必须支持鼠标与键盘。
5. **逻辑可测试**：游戏规则与视觉层解耦，随机行为可注入或固定种子复现。

## MVP Requirements

### R1. 游戏入口与单局流程

- 用户进入页面后可直接开始一局，不出现登录、广告或付费门槛。
- 完整流程包含：开局 → 选择/交换 → 匹配消除 → 连锁结算 → 达成目标或耗尽步数 → 结果 → 再来一局。
- 页面刷新后无需恢复未完成对局；本地最佳成绩可保留。

### R2. 经典三消规则

- 棋盘由规则网格和至少 6 种视觉可区分的普通棋子组成。
- 只能交换上下左右相邻棋子；交换后未形成有效匹配时自动复位。
- 横向或纵向连续 3 个及以上同类棋子触发消除；交叉匹配中的公共棋子只结算一次。
- 消除后棋子向下坠落、顶部补充，并持续结算由坠落产生的连锁。
- 初始棋盘不得直接存在可消除组合，且至少存在一个合法移动。
- 无合法移动时自动洗牌；洗牌不消耗步数且不能产生初始自动消除。

### R3. 目标、计分与结果

- 首版采用“限定步数内达到目标分”的单一模式。
- 有效交换才消耗步数；无效交换不消耗步数。
- 基础消除、4 个及以上长连、交叉组合和连续坠落需要产生可解释的加分差异。
- 局内持续展示当前分数、目标分、剩余步数和当前/最高连击。
- 达成目标立即判定胜利；步数耗尽且未达标则判定失败。

### R4. 交互与反馈

- 支持点击/轻触选择两个相邻棋子；桌面端同时支持键盘完成等价操作。
- 选择、非法交换、有效交换、消除、坠落、连锁、洗牌、胜利和失败均有明确反馈。
- 动效期间必须锁定冲突输入，避免棋盘状态重入或重复结算。
- 支持 `prefers-reduced-motion`，减少动态时仍能理解状态变化。

### R5. 响应式与可访问性

- 覆盖 320px 宽手机、常见 390px 手机、平板与桌面视口，无横向溢出。
- 棋盘在手机首屏中保持完整可操作，核心触控目标不小于 44×44 CSS 像素，或通过等效间距满足误触防护。
- 不仅依赖颜色区分棋子；每种棋子还需有不同形状、纹理或图案。
- 提供可读的按钮名称、焦点状态、状态播报与合理标题层级。

### R6. 技术与数据边界

- 前端采用 Vue 3、Vite、TypeScript 和 pnpm workspace；MVP 不引入后端。
- 游戏状态默认仅存在浏览器内存；最佳成绩使用版本化本地存储。
- 核心规则不得依赖 DOM 或动画时间，便于纯函数测试。
- 不接入账号、排行榜、社交分享、支付、广告、埋点或远程配置。

## Acceptance Criteria

- [ ] AC1：用户在手机或桌面浏览器中打开页面后，无需账号即可完成一局从开局到结算的游戏。
- [ ] AC2：相邻交换、无效复位、3+ 匹配、交叉去重、坠落补充、连锁和无解洗牌均符合需求并可通过确定性测试复现。
- [ ] AC3：只有有效交换消耗步数，分数与连击在每次结算后准确更新，胜负条件不会重复触发。
- [ ] AC4：320px、390px、768px 与 1440px 视口均无横向溢出或不可点击棋子。
- [ ] AC5：鼠标、触控与键盘均可完成主要游戏操作；减少动态模式下功能完整。
- [ ] AC6：棋子不只通过颜色区分，核心状态具备可访问名称和播报。
- [ ] AC7：核心规则单元测试、关键流程浏览器测试、类型检查、lint 和生产构建全部通过。
- [ ] AC8：MVP 不包含后端、账号、远程排行榜、支付、广告或与 ParallelLines 品牌耦合的内容。

## Out of Scope

- 关卡地图、体力系统、每日任务和长期养成。
- 炸弹、彩虹球等特殊棋子与复杂组合技。
- 登录注册、云端存档、跨设备同步和在线排行榜。
- 多人对战、社交分享、聊天、好友与公会。
- 原生 App、PWA 离线安装、推送通知。
- 商业化、广告、支付和数据埋点。
