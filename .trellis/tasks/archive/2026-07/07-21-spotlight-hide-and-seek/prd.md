# 探照灯捉迷藏素材模式

## Goal

把当前右下角直接可见的小鱼区域改造成低暴露、可搜索的桌面捉迷藏体验：小鱼默认隐藏并以散落与局部聚拢混合方式分布在更大的桌面范围，玩家用探照灯发现它们，选择送入托盘，或拖给小猫直接吃掉。

## Background

- 产品使用八种毛毡小鱼；`07-21-rename-jelly-domain-to-fish` 在本任务前完成内部类型、组件、样式、存档边界和文案的领域统一。
- 当前实现把素材限制在右下角最多约 `520 × 300px` 的容器，通过 `spread` 与 `pile` 坐标在闲置散开和交互聚拢间切换。
- 当前遮挡、可选择状态和方向键导航依赖 `pile` 坐标；全桌面混合布局必须保持视觉堆叠和逻辑遮挡一致。
- 现有产品契约要求指针、触屏、完整键盘、屏幕阅读器、320px 视口、减少动态效果和 Document Picture-in-Picture 均可使用。
- 小猫投喂任务已经定义每堆最多投喂三条、第三条后吃饱睡觉、下一堆重置，以及任意合法投喂后关卡仍可完成。

## Requirements

### R1 — Mixed hidden fish field

- 初始布局直接生成稳定的混合状态：部分小鱼单条或疏散分布，部分形成不规则局部鱼堆；不再用注意力触发整场散开/聚拢切换。
- 搜索范围明显超出当前右下角容器，并避开小猫休息位、植物、可见托盘、安静控制和安全区边距。
- 探照灯只控制显隐，不改变小鱼位置、分组、层级或遮挡。
- 每堆同时包含可单独发现的散落小鱼和需要从上层逐条取出的局部鱼堆；视觉覆盖必须与可选择规则一致。

### R2 — Spotlight reveal

- 桌面细指针设备在未搜索时看不到可辨认的小鱼；指针附近的小鱼平滑显影，离开范围后重新隐藏。
- 探照灯只作用于小鱼及必要交互反馈，不在背景上绘制明显的手电筒圆斑或游戏 HUD。
- 拖拽中的小鱼脱离探照灯后仍保持可见；减少动态效果模式可取消光圈、余光和移动补间，但保留即时显影边界与状态反馈。

### R3 — Selection and feeding

- 发现小鱼后，普通点击沿用现有规则送入七格托盘，不需要先执行召集动作。
- 只有当前未被上层遮挡、可选择的小鱼能开始拖拽或投喂。
- 拖到小猫当前接收区会直接移除小鱼，不进入托盘、不参与凑三；释放在其他位置或投喂被拒绝时稳定回到原位置。
- 每堆最多投喂三条；第三条后小猫吃饱、躺下并睡觉，第四次投喂被拒绝；下一堆生成时恢复容量。
- 任意合法的最多三次投喂都不能让剩余关卡无法完成。
- 投喂只推进小猫饱腹与姿态。被吃掉的小鱼参与当前场景清空判断，但不增加 `clearCount`，不推动植物成长；只有三条同类消除推动植物。

### R4 — Cat collaboration

- 小猫和植物作为桌面摆件常驻可见。空闲小猫会低频自动做状态合适的卖萌动作，但绝不自动定位、显影或靠近隐藏小鱼。
- 单击、触摸或键盘激活清醒且未吃饱的小猫，会请求它寻找一条当前可选择但尚未显影的小鱼。
- 寻鱼只提供场景内位置提示，不自动显影、选择、移除或投喂目标。
- 小猫抵达目标后停在附近守候，通过朝向或轻拍提示位置，直到目标进入托盘、被投喂或因其他合法状态变化失效。
- 守候中的小猫位置是有效投喂接收区；目标处理完成后小猫返回固定休息位。
- 自动卖萌、重复激活和布局动画不能打断守候或丢失目标；吃饱、躺下或睡眠时拒绝寻鱼并给出状态合适的反馈。
- 页面失焦或隐藏时暂停自动反应与移动计时，不累计或补播离开期间的动作。

### R5 — Input and accessibility

- 鼠标探照灯随指针移动。
- 触屏按住时以接触点形成探照灯并可拖动搜索；松手后保留短暂余光再隐藏，同时避免与拖鱼投喂和页面手势误触。
- 键盘聚焦搜索桌面后用方向键移动探照灯，Enter/Space 选择显影且可选择的小鱼；Tab 可到达小猫和安静控制，焦点始终可见。
- 屏幕阅读器不必执行视觉空间搜索，可直接获知并操作当前可选择的小鱼，同时遵守相同的遮挡、入托盘和投喂规则。
- 减少动态效果模式使用即时状态变化，不能取消功能、焦点或文本反馈。

### R6 — Idle visibility hierarchy

- 小鱼默认完全隐藏，小猫和植物常驻可见。
- 空托盘近乎隐去；第一条小鱼进入后恢复到可读状态，刷新后依据已恢复托盘状态正确显示。
- 静音和小窗控制低透明待机，在指针、触摸或键盘接近时恢复可发现、可操作状态。

### R7 — State and compatibility

- 混合布局、投喂次数、小猫饱腹/姿态及必要守候目标随规范游戏快照恢复。
- 指针位置、触摸余光、拖拽过程和临时卖萌动作不持久化。
- 现有有效快照通过领域迁移边界安全加载，不清空用户进度；损坏或失效的守候目标安全回到小猫休息位。
- 视口变化和 Picture-in-Picture 移动重新投影同一布局，不重新生成或改变当前小鱼。

## Acceptance Criteria

- [x] AC1：未搜索时无可辨认小鱼；鼠标、触屏和键盘探照灯只显影局部小鱼，离开/余光结束后重新隐藏。
- [x] AC2：每堆包含散落单鱼与局部鱼堆，覆盖全桌面安全区域，显影不改变位置或遮挡关系。
- [x] AC3：被发现且可选择的小鱼可点击入托盘；隐藏或被遮挡的小鱼不会误接收指针操作。
- [x] AC4：可选择小鱼可拖给小猫直接移除；失败/拒绝拖放回原位，托盘不变。
- [x] AC5：每堆仅前三次投喂成功，第四次被拒绝；投喂不增加 `clearCount` 或植物成长，且剩余关卡仍可完成。
- [x] AC6：激活清醒小猫会寻找并守候一条合法隐藏目标；目标处理后返回，吃饱/睡眠时拒绝寻找。
- [x] AC7：自动卖萌低频、非阻塞、不找鱼；失焦/隐藏时暂停且返回后不补播。
- [x] AC8：守候位置可接收拖拽投喂，且目标绑定不会被重复激活、自动反应或动画破坏。
- [x] AC9：空托盘近乎不可见，非空托盘清晰；小猫、植物常驻，安静控制待机弱化但可发现。
- [x] AC10：320px、触屏、键盘、屏幕阅读器和减少动态效果模式均可完成选择、投喂与消除。
- [x] AC11：有效旧快照、刷新、视口变化和 Picture-in-Picture 保持可玩状态、规范布局、托盘与小猫状态。
- [x] AC12：定向引擎、存档、控制器、组件测试及前端 lint、type-check、build 通过，响应式视觉 QA 无横向滚动或关键遮挡。

## Out of Scope

- 计时、得分、任务、地图、提示箭头或显式“还剩几条”的 HUD。
- 自动找鱼、自动选择、自动投喂，或因离开页面产生的奖励、惩罚和状态推进。
- 系统级置顶、桌面覆盖层、浏览器扩展或原生桌宠。
- 借本任务重新设计匹配算法、植物成长里程碑或八种鱼素材本身。
