# 局内目标与本地进度

## Goal

为核心三消规则增加玩家姓名、可解释的单局目标、计分、步数、胜负、
本地最佳成绩和本机排行榜，使 MVP 形成完整且可重复游玩的挑战闭环。

## Requirements

### R1. 对局状态

- 打开页面后先显示姓名输入；姓名校验通过前不能开始操作棋盘。
- 姓名去除首尾空白并折叠连续空白，长度为 1–12 个 Unicode 字符。
- 对局状态至少包含：等待玩家、进行中、结算中、胜利、失败。
- 状态转换必须单向且可追踪；同一对局不能重复扣步、重复计分或重复结算。
- 同名再来一局重置棋盘、当前分数、剩余步数和连击；换玩家返回姓名输入，
  两种路径都保留本地最佳成绩和排行榜。

### R2. 步数

- 每次产生至少一组匹配的有效交换消耗 1 步。
- 非相邻、越界、同格和无匹配复位均不消耗步数。
- 洗牌和自动连锁不额外消耗步数。
- 剩余步数为 0 后禁止继续交换，并立即进入胜负判定。

### R3. 计分

- 计分规则必须集中配置并可通过 UI 文案解释。
- 基础三连获得基础分；4 个及以上长连、同轮多个匹配组和连续坠落获得额外奖励。
- 交叉匹配中的公共棋子不重复计算基础棋子分，但可获得组合奖励。
- 连锁倍率按自动结算轮次增加，并在新的用户交换开始时重置。
- 分数使用安全整数范围，显示层负责格式化，不在核心计分中混入本地化字符串。

### R4. 目标与难度

- MVP 使用“限定步数内达到目标分”的模式。
- 初始步数与目标分作为配置项，不散落在组件中。
- 目标分需要基于固定随机种子模拟或人工试玩校准，使常规玩家具有合理成功率；不得仅凭视觉效果写死。
- 若玩家在步数耗尽前达到目标分，立即胜利，不要求消耗完剩余步数。

### R5. 本地最佳成绩

- 保存历史最高分、最佳连击和最近完成时间；不保存完整棋盘或未完成对局。
- 使用带版本号和运行时校验的本地存储结构；损坏、旧版或不可访问存储不得阻止游戏启动。
- 仅保存用户主动输入的本地显示姓名；不保存设备指纹、远程标识或其他身份信息，
  不向任何服务器发送姓名或成绩。

### R6. 结果与重开

- 胜利和失败结果展示玩家姓名、本局得分、目标差距/剩余步数、最高连击、
  排行名次和最佳成绩变化。
- 从结果层可用同一姓名开始新局，或返回姓名输入切换玩家。
- 进行中重开需防止误触；确认后生成独立新对局，不沿用旧棋盘状态。

### R7. 本机排行榜

- 每局胜利或失败只写入一次近期历史，记录玩家姓名、分数、最高连击、完成时间和结果。
- 仅提供本周、本月、总排行三个本机榜单；本周从当地时间周一 00:00 开始，
  本月从当地时间当月 1 日 00:00 开始。
- 每个榜单按姓名只保留该周期的最佳成绩，最多展示 10 名；按分数降序、
  最高连击降序、完成时间升序稳定排序。
- 排行榜前三名重点展示，其余名次使用紧凑列表；所有玩家使用同一个本地默认头像，
  当前玩家的名次需要高亮。
- 页面持续展示当前选择的本机榜单；没有记录时解释完成一局后会出现成绩。
- 排行榜与最佳成绩共用同一版本化存储边界，损坏条目必须被整体安全丢弃。

## Acceptance Criteria

- [x] AC1：姓名为空、纯空白或超过 12 个字符时不能开始；有效姓名规范化后进入对局。
- [x] AC2：只有有效交换扣除 1 步，无效交换、连锁和洗牌不扣步。
- [x] AC3：三连、长连、多组匹配、交叉匹配和多轮连锁均按文档规则得到确定分数。
- [x] AC4：连锁倍率在同一结算链中递增，在下一次玩家交换时重置。
- [x] AC5：达到目标时只触发一次胜利；步数为 0 且未达标时只触发一次失败。
- [x] AC6：同名新局与换玩家路径正确重置局内状态，并在进行中重开时防止误触。
- [x] AC7：有效本地记录可恢复；损坏、过期或禁止访问的本地存储会安全回退。
- [x] AC8：本周、本月、总排行只使用已完成对局，每个周期按玩家最佳成绩稳定选出
  前 10 名；默认头像、当前玩家高亮和空态正确，姓名和成绩不离开浏览器。
- [x] AC9：固定种子模拟支持最终目标分选择，并将结果写入任务设计说明。

## Dependencies

- 依赖 `07-17-match3-core-engine` 的匹配组、消除数量和连锁轮次输出。
- 与 `07-17-playable-game-ui` 约定 HUD、结果层和重开确认所需状态。

## Out of Scope

- 多关卡难度曲线、星级、成就、每日任务和体力。
- 云端存档、跨设备同步、账号系统和在线排行榜。
- 好友榜、赛季榜、赛季徽章、排行倒计时和社交关系。
- 道具、货币、商店和商业化。
